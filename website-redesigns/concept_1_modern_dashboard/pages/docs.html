Getting Started

ODB+ connects to your vehicle through an OBD-II diagnostic adapter. Here is what you need to get started:

Requirements

  • Android device running Android 8.0 (Oreo) or later
  • An ELM327-compatible OBD-II Bluetooth or WiFi adapter
  • A 1996 or newer vehicle with an OBD-II diagnostic port (all US vehicles 1996+, EU vehicles 2001+)

Installation

  1. Download ODB+ from the Google Play Store or from our website
  2. Open the app and grant Bluetooth permissions when prompted
  3. Plug your OBD-II adapter into your vehicle's diagnostic port (usually located under the dashboard on the driver's side)
  4. Turn on your vehicle's ignition (engine running is recommended for live data)

Quick Start

1. Open ODB+ and tap "Connect"
2. Select your adapter from the Bluetooth list
3. Wait for protocol negotiation (~2-5 seconds)
4. Start reading data!

Adapter Setup

ODB+ supports any ELM327-compatible adapter over Bluetooth Classic or WiFi. Here is what you should know about choosing and configuring an adapter.

Recommended Adapters

ODB+ uses the Universal OBD-II Adapter Protocol Layer (UOAPL) to automatically detect and configure your adapter. The following adapter families are supported:

  • ELM327 -- The most common and widely available. Look for genuine v1.5 or v2.1+ chips. Avoid cheap clones labeled "v2.1" that are actually v1.0 internals.
  • STN11xx/STN21xx -- Higher performance adapters from OBDLink. Faster command throughput and better CAN support.
  • ESP32-based -- Custom adapters using ESP32 microcontrollers. Used by some open-source projects.

Bluetooth Pairing

  1. Go to your Android device's Bluetooth settings
  2. Ensure your OBD-II adapter is plugged in and the vehicle ignition is on
  3. Scan for devices -- your adapter will appear as something like "OBD-II", "OBDII", "Android-Vlink", or "ELM327"
  4. Pair with the default PIN: 1234 or 0000
  5. Return to ODB+ and tap Connect

WiFi Adapters

For WiFi-based adapters, connect to the adapter's WiFi network first (usually named something like "WiFi_OBDII" or "CLinkV200"), then open ODB+ and connect via TCP. The default address is 192.168.0.10:35000.

Troubleshooting Adapters

# If connection fails, try these steps in order:
1. Unplug and re-plug the adapter
2. Turn vehicle ignition off, wait 10s, turn back on
3. Remove Bluetooth pairing, re-pair the device
4. In ODB+, go to Terminal and send: ATZ
   - Should respond: "ELM327 v2.x" or similar
5. Try: ATDP
   - Shows detected protocol

Connecting to Your Vehicle

When you tap Connect in ODB+, here is what happens behind the scenes:

  1. Transport layer -- Establishes Bluetooth or TCP socket connection to the adapter hardware
  2. Adapter fingerprinting -- Identifies the adapter family (ELM327, STN, ESP32) and firmware version
  3. Driver initialization -- Configures the adapter with optimal settings (echo off, line feeds off, headers on)
  4. Protocol negotiation -- Discovers which OBD-II protocol your vehicle uses (CAN, KWP2000, ISO 9141, J1850)
  5. PID discovery -- Queries the ECU for supported parameter IDs and builds a list of available sensors

The entire process typically takes 3-8 seconds depending on your vehicle and adapter.

Supported Protocols

  • CAN (ISO 15765-4) -- Used by most vehicles 2008+. 11-bit and 29-bit addressing supported.
  • KWP2000 Fast Init (ISO 14230-4) -- Common in European and Asian vehicles 2001-2010.
  • ISO 9141-2 -- Older European vehicles, uses 5-baud initialization.
  • SAE J1850 PWM -- Primarily Ford vehicles pre-2008.
  • SAE J1850 VPW -- Primarily GM vehicles pre-2008.

Reading Live Data

Once connected, navigate to the Live Data screen to see real-time sensor readings. ODB+ automatically discovers which PIDs your vehicle supports and displays them in a scrollable list.

Understanding PID Values

Each parameter ID (PID) corresponds to a specific sensor or calculated value from the ECU:

  • 010C -- Engine RPM (0-16,383 rev/min)
  • 010D -- Vehicle Speed (0-255 km/h)
  • 0105 -- Engine Coolant Temperature (-40 to 215 C)
  • 0110 -- Mass Air Flow Rate (0-655.35 g/s)
  • 0106 -- Short Term Fuel Trim (-100% to +99.2%)
  • 010B -- Intake Manifold Pressure (0-255 kPa)

Customizing the Display

Tap the gear icon on the Live Data screen to customize which PIDs are displayed, their order, and the display format (gauge view, list view, or graph view). You can create multiple dashboard layouts and switch between them.

Trouble Codes

The DTC screen reads diagnostic trouble codes from your vehicle's ECU. Codes are organized into three categories:

  • Stored/Active -- Currently triggering the check engine light (Mode 03)
  • Pending -- Detected but not yet confirmed after multiple drive cycles (Mode 07)
  • Permanent -- Codes that remain until the issue is verified fixed by the ECU (Mode 0A)

Clearing Codes

You can clear stored and pending codes by tapping the "Clear Codes" button. This sends Mode 04 to the ECU, which:

  • Clears all stored DTCs
  • Resets the MIL (check engine light)
  • Clears all freeze frame data
  • Resets all readiness monitors to "not complete"

Note: Clearing codes does not fix the underlying problem. If the issue persists, the code will return after one or more drive cycles.

Terminal Mode

Terminal mode provides direct command-line access to the OBD-II adapter. You can send any AT command or OBD-II PID request and see the raw response.

Common Commands

# Adapter info
ATZ          # Reset adapter
ATI          # Print firmware version
ATDP         # Display current protocol
ATRV         # Read vehicle voltage

# OBD-II queries
0100         # Supported PIDs (01-20)
0120         # Supported PIDs (21-40)
0140         # Supported PIDs (41-60)
010C         # Engine RPM
010D         # Vehicle speed
0105         # Coolant temperature
03           # Read stored DTCs
07           # Read pending DTCs
04           # Clear DTCs (use with caution)

# Configuration
ATE0         # Echo off
ATH1         # Headers on (shows protocol headers)
ATS1         # Spaces on (readable hex format)
ATSP0        # Auto-detect protocol

API Reference

ODB+ provides an intent-based API for other Android apps to request diagnostic data. This is useful for integration with fleet management tools, custom dashboards, or data logging applications.

Requesting Data

// Send an intent to request live data
val intent = Intent("com.odbplus.ACTION_READ_PID")
intent.putExtra("pid", "010C") // Engine RPM
intent.setPackage("com.odbplus.app")
startActivityForResult(intent, REQUEST_PID)

Receiving Data

override fun onActivityResult(requestCode: Int, resultCode: Int, data: Intent?) {
    if (requestCode == REQUEST_PID && resultCode == RESULT_OK) {
        val pidValue = data?.getFloatExtra("value", 0f)
        val pidUnit = data?.getStringExtra("unit")
        val pidName = data?.getStringExtra("name")
        // pidValue = 2847.0, pidUnit = "RPM", pidName = "Engine RPM"
    }
}

Available Intents

  • com.odbplus.ACTION_READ_PID -- Read a single PID value
  • com.odbplus.ACTION_READ_DTCS -- Read all stored trouble codes
  • com.odbplus.ACTION_CLEAR_DTCS -- Clear trouble codes (requires user confirmation)
  • com.odbplus.ACTION_CONNECTION_STATUS -- Check if ODB+ is connected

Frequently Asked Questions

What adapters work with ODB+?

Any ELM327-compatible Bluetooth or WiFi OBD-II adapter. We recommend adapters with genuine ELM327 v1.5+ or STN11xx chips. Avoid ultra-cheap adapters from unbranded sellers -- many advertise "v2.1" but contain v1.0 firmware that lacks support for some protocols.

Why does it say "UNABLE TO CONNECT"?

This usually means the adapter cannot establish communication with your vehicle's ECU. Common causes:

  • Ignition is not turned on (must be in ON or RUN position)
  • Adapter is not fully seated in the OBD-II port
  • Vehicle uses a protocol that requires a specific initialization sequence (ODB+ handles this automatically, but some very old vehicles may need manual protocol selection)

Does ODB+ work with diesel vehicles?

Yes. ODB+ supports all OBD-II compliant vehicles regardless of fuel type, including diesel, gasoline, hybrid, and electric vehicles. Some diesel-specific PIDs (like DPF pressure or EGR rate) are available if supported by the vehicle.

Can I use ODB+ while driving?

Yes, but safety first. Use the HUD (heads-up display) mode which provides a simplified overlay, or have a passenger operate the app. Never interact with the full app interface while driving.

Will ODB+ void my warranty?

No. ODB+ only reads data from your vehicle's ECU using standardized OBD-II commands. It does not modify any ECU parameters, reflash firmware, or alter vehicle settings. Reading diagnostic data is the same thing your dealership's scan tool does.

Is my vehicle data stored or shared?

All diagnostic data stays on your device. ODB+ does not upload vehicle data to any server. Session logs are stored locally and can only be shared if you explicitly export them. The AI diagnostics feature sends anonymized sensor snapshots for analysis but never VINs or location data.
