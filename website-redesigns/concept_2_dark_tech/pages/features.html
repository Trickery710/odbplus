01 AI Engine

AI-Powered Diagnostics

Chat with an AI that has direct access to your vehicle's live sensor data. Describe a symptom -- "my car shakes at idle" -- and the AI correlates it with real-time RPM variance, misfire counts, fuel trim drift, and O2 sensor patterns to give you a targeted diagnosis.

Unlike generic chatbots, ODB+'s AI sees the actual numbers. It knows your coolant temp is 105C when it should be 90C. It sees your short-term fuel trim at +18% when normal is +/-5%. It diagnoses from data, not guesses.

02 Real-Time

Live Data Monitoring

Stream real-time sensor data from your vehicle's ECU. Monitor RPM, engine load, coolant and intake air temperatures, O2 sensor voltages, mass air flow rate, manifold absolute pressure, fuel trims, timing advance, and dozens more PIDs.

Configurable refresh rates let you balance data density against adapter bandwidth. Set threshold alerts to flag abnormal readings. View data as scrolling values, live graphs, or a full-screen gauge HUD.

OBD-II Commands

// Query O2 sensor Bank 1 Sensor 1

03 Database

18,000+ Error Codes

Comprehensive DTC database covering all standard OBD-II generic codes (P0xxx, P2xxx), manufacturer-specific codes (P1xxx), body codes (Bxxxx), chassis codes (Cxxxx), and network codes (Uxxxx).

Each code entry includes a plain-language description, severity level, common causes ranked by probability, affected components, recommended diagnostic steps, and estimated repair cost ranges.

04 Connectivity

Bluetooth OBD-II Support

Connect wirelessly via Bluetooth Classic or WiFi to any ELM327-compatible adapter. ODB+ also supports STN-series and ESP32-based adapters with enhanced capabilities through its Universal OBD-II Adapter Protocol Layer (UOAPL).

The adapter fingerprinting system automatically identifies your adapter's capabilities, selects the optimal driver, and configures communication parameters. No manual setup required.

09 Reference

Sensor Reference Library

Comprehensive reference guides for every common automotive sensor. Learn what each sensor does, where it is located, normal operating ranges, symptoms of failure, and how to test it with a multimeter.

Cross-referenced with OBD-II PIDs so you can jump from a live reading directly to the sensor reference for that parameter.

Sensor: O2 Sensor (Bank 1, Sensor 1)

OXYGEN SENSOR (LAMBDA)

PID: 0x14

Location: Exhaust manifold, pre-catalytic converter

Type: Narrowband zirconia (0-1V)

Normal range: 0.1V - 0.9V (oscillating)

Failure symptoms: Poor fuel economy, rough idle, failed emissions

TEST: Sensor should oscillate between 0.1V and 0.9V at idle. Stuck low = lean exhaust. Stuck high = rich exhaust.
