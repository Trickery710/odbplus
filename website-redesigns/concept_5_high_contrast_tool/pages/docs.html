GETTING STARTED

REQUIREMENTS

COMPONENT REQUIREMENT
ANDROID VERSION 8.0+ (API 26)
BLUETOOTH BLUETOOTH CLASSIC (SPP) OR BLUETOOTH LE
OBD ADAPTER ELM327 V1.0+ COMPATIBLE
VEHICLE OBD-II COMPLIANT (1996+ US, 2001+ EU)
STORAGE 50 MB FREE SPACE

QUICK START SEQUENCE

1. Install ODB+ from Google Play
2. Plug OBD-II adapter into vehicle diagnostic port
3. Turn vehicle ignition to ON (engine running or ACC)
4. Open ODB+ > Settings > Bluetooth > Select adapter
5. Tap CONNECT on main screen
6. Wait for protocol negotiation (~2-30 seconds)
7. System ready. Begin diagnostics.

The app will automatically detect your adapter type (ELM327, STN, ESP32) and negotiate the correct vehicle protocol. No manual configuration required for most vehicles.

BLUETOOTH PAIRING

> Android Settings > Bluetooth > Pair New Device
> Select adapter (common names: "OBDII", "Android-Vlink", "OBDLink MX+")
> PIN: 1234 (default for most ELM327 adapters)
> Return to ODB+ > Settings > Connection > Select paired device
> Tap CONNECT

If pairing fails, power-cycle the adapter by unplugging it from the OBD port for 10 seconds. Some adapters require the vehicle ignition to be ON during pairing.

AT COMMAND REFERENCE

These commands can be sent directly via Terminal Mode. AT commands configure the OBD adapter. OBD commands (hex values) query the vehicle ECU.

ADAPTER CONFIGURATION

ATE0 Echo off (disable command echo) OK
ATE1 Echo on (enable command echo) OK
ATL0 Linefeeds off OK
ATS0 Spaces off in responses OK
ATS1 Spaces on in responses (recommended) OK
ATH0 Headers off OK
ATH1 Headers on (show protocol headers) OK
ATSP0 Set protocol to automatic OK
ATSP5 Set protocol to KWP2000 fast init OK
ATSP6 Set protocol to CAN 11-bit 500kbps OK

ATRV Read battery voltage 12.6V

PROTOCOL NEGOTIATION NOTES

> RECOMMENDED: Do NOT send ATSP0 first on KWP2000 vehicles.
> The ELM retains last-used protocol after ATZ.
> Send 0100 directly to use cached protocol hint.
> If 0100 fails, fall back to protocol-specific ATSP commands.
> Fallback order: ATSP5 (KWP fast) BEFORE ATSP3 (ISO 9141).
> ISO 5-baud init = ~2.7s x 3 retries. KWP fast = ~200ms.
> Wrong order can delay connection past 30s timeout.

TROUBLESHOOTING

COMMON ISSUES

SYMPTOM CAUSE SOLUTION
UNABLE TO CONNECT Adapter not powered / wrong Bluetooth device Verify adapter LED is on. Re-pair Bluetooth. Check OBD port fuse.
NO DATA on all PIDs Protocol mismatch / ATCAF0 active Send ATZ to reset. Verify ATCAF1 is set. Try ATSP0.
BUS INIT: ERROR Wrong protocol / corrupted headers Send ATZ, then ATSP5 for KWP or ATSP6 for CAN. Do not send ATSH commands on ISO protocols.
Slow response (>5s) ISO 9141 5-baud init / wrong protocol order Verify correct protocol. KWP fast (ATSP5) should be tried before ISO 9141 (ATSP3).
Response too short: 0 bytes ATS0 active / unspaced responses Send ATS1 to enable spaces. Required for KWP2000 response parsing.
Adapter disconnects Bluetooth timeout / power saving Disable Android battery optimization for ODB+. Move closer to adapter.
