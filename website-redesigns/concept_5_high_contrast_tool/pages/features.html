SYSTEM REGISTRY

FEATURE SPECIFICATIONS

> Complete module manifest. Each subsystem documented with technical parameters.

MODULE 01

AI-Powered Diagnostics

ACTIVE
DESCRIPTION Conversational AI assistant with real-time access to vehicle sensor data. Analyzes live PID streams, interprets DTCs, and provides manufacturer-specific repair guidance.
AI MODEL GPT-4 class large language model with automotive fine-tuning
DATA CONTEXT Live PID values, active DTCs, freeze frame data, vehicle VIN/make/model/year
CAPABILITIES Root cause analysis, repair cost estimation, parts identification, severity assessment, TSB cross-reference
RESPONSE TIME 1-3 seconds average
OFFLINE MODE Basic DTC lookup available offline. AI chat requires network connection.
LANGUAGES English, Spanish, German, French, Portuguese, Japanese
MODULE 02

Live Data Monitoring

ACTIVE
DESCRIPTION Real-time streaming of all supported OBD-II PIDs from the vehicle ECU. Multiple display modes: table, graph, gauge cluster, and raw hex.
REFRESH RATE Up to 60Hz on CAN bus adapters. 10-20Hz on ISO/KWP protocols.
SUPPORTED PIDS 200+ across Mode 01 (live), Mode 02 (freeze frame), Mode 09 (vehicle info)
DISPLAY MODES Tabular readout, real-time line graphs, radial gauge cluster, HUD overlay, raw hex terminal
DATA LOGGING Continuous background recording. Export CSV/JSON. Timestamped to millisecond precision.
ALERTING Configurable high/low threshold alerts per PID. Audio and visual notification.
KEY SENSORS RPM, vehicle speed, coolant temp, intake temp, MAF, MAP, O2 sensors (all banks), fuel trims (short/long), timing advance, engine load, throttle position, battery voltage
MODULE 03

Error Code Database

ACTIVE
TOTAL CODES 18,247 diagnostic trouble codes
CATEGORIES P0xxx (generic powertrain), P1xxx (mfr-specific), P2xxx (generic ISO/SAE), Bxxxx (body), Cxxxx (chassis), Uxxxx (network)
MANUFACTURERS 50+ brands: Toyota, Honda, Ford, GM, Chrysler/Stellantis, BMW, Mercedes, Audi/VW, Nissan, Hyundai/Kia, Subaru, Mazda, Mitsubishi, Volvo, Jaguar/Land Rover, Porsche, and more
CODE DETAIL Full description, affected system, severity rating, common causes, symptoms, repair procedures, estimated labor time, related TSBs
SCAN MODES Mode 03 (stored DTCs), Mode 07 (pending DTCs), Mode 0A (permanent DTCs)
CLEAR CODES Mode 04 clear/reset. Confirmation required. Warning displayed for emissions-related codes.
FREEZE FRAME Mode 02 freeze frame data retrieval for each stored DTC
MODULE 04

Terminal Mode

ACTIVE
DESCRIPTION Raw command-line interface to the OBD-II adapter. Full AT command support. Custom PID queries. Protocol-level debugging.
COMMAND SET Full ELM327 AT command set, STN extended commands, raw OBD-II service requests (Mode 01-0A)
DISPLAY Monospace terminal with syntax highlighting. Hex and ASCII view modes. Auto-parsed responses.
HISTORY Persistent command history. Favorites/bookmarks. Macro support for command sequences.
TIMEOUT Configurable response timeout (100ms - 30s). Auto-retry on NO DATA.
PROTOCOLS Manual protocol selection (ATSP0-ATSP9) or automatic negotiation
MODULE 05

Session Logger & Replay

ACTIVE
DESCRIPTION Records every PID reading, DTC scan, and terminal command during a diagnostic session. Full replay capability.
RECORDING Automatic start on connection. Manual start/stop. Background recording.
REPLAY Time-scrubbing playback. Variable speed (0.5x - 10x). Pause/resume. PID value overlay.
EXPORT FORMATS CSV (spreadsheet), JSON (programmatic), PDF (report), raw log
STORAGE Local device storage. Cloud sync optional. Typical session: 2-5 MB.
SHARING Share session files via email, messaging, or direct mechanic link
MODULE 06

Guided Diagnostic Tests

ACTIVE
DESCRIPTION Step-by-step automated test procedures with real-time pass/fail evaluation. Instructor-guided or fully automatic.
AVAILABLE TESTS RPM stability, O2 sensor response time, catalytic converter efficiency, fuel system feedback, EVAP system integrity, misfire detection, idle quality
HUD MODE Full-screen heads-up display overlay. Large readable values for under-hood use. Auto-brightness.
RESULTS PASS / FAIL / WARNING with detailed explanation. Historical comparison against previous tests.
CUSTOM TESTS Create custom test sequences with user-defined PID thresholds and timing
MODULE 07

Vehicle Forums

ACTIVE
DESCRIPTION Manufacturer-organized community forums. Share diagnostic data, discuss repairs, get advice from experienced mechanics.
MANUFACTURERS 50+ dedicated sub-forums organized by brand, model, and year range
FEATURES DTC auto-linking, session attachment, image uploads, reputation system, verified mechanic badges
SEARCH Full-text search across all posts. Filter by DTC code, vehicle, date, solved status.
MODULE 08

Sensor Reference Library

ACTIVE
DESCRIPTION Comprehensive reference for every OBD-II sensor type. Pinouts, expected values, testing procedures, failure modes.
SENSOR TYPES O2 (narrowband/wideband), MAP, MAF, TPS, CKP, CMP, ECT, IAT, knock, EGR position, fuel pressure, boost pressure
DATA INCLUDED Normal operating ranges, waveform patterns, connector pinouts, resistance values, voltage specifications
TEST GUIDES Step-by-step testing procedures using multimeter, oscilloscope, and OBD-II scan data
MODULE 09

Parts Catalog

ACTIVE
DESCRIPTION Integrated parts lookup linked to diagnostic results. When a DTC points to a failed component, find replacement parts immediately.
SOURCES OEM part numbers, aftermarket cross-references, pricing from multiple vendors
INTEGRATION Auto-linked from DTC repair guides. Filter by vehicle year/make/model.
PRICING Real-time pricing comparison across vendors. Price history tracking.
MODULE 10

Adapter Compatibility

ACTIVE
DESCRIPTION Universal OBD-II adapter support via UOAPL (Universal OBD-II Adapter Protocol Layer). Automatic adapter fingerprinting and driver selection.
ADAPTER FAMILIES ELM327 (v1.0-2.3+), STN1110/STN1170, STN2100/STN2120, ESP32-based, OBDLink (MX/MX+/EX/SX)
CONNECTION Bluetooth Classic (SPP), Bluetooth LE, WiFi (TCP socket)
FINGERPRINTING Automatic adapter identification: firmware version, supported AT commands, CAN filter capability, header support
PROTOCOLS SAE J1850 PWM, SAE J1850 VPW, ISO 9141-2, ISO 14230-4 (KWP2000), ISO 15765-4 (CAN 11/29-bit)
NEGOTIATION Automatic protocol detection with optimized fallback order. Fast-path detection reuses last successful protocol.
